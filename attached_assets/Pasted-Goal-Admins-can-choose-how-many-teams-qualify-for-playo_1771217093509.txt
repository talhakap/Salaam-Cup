Goal:
Admins can choose how many teams qualify for playoffs (ex: 2–128). The system auto-generates a single-elimination bracket, auto-fills seeds from standings, auto-advances winners as scores are entered, and still allows admin overrides (swap teams, change seeds, lock bracket).

KEY REQUIREMENTS

1) Admin Setup UI (Playoffs Settings)
Create an admin page: /admin/playoffs/:tournamentId/:divisionId

Controls:
- “Teams that qualify” (qualifyCount): integer. Default options: 2, 3, 4, 5, 6, 7, 8, 10, 12, 16, 24, 32, 48, 64. Also allow “custom” numeric input.
- “Bracket mode”: Byes or Play-in (same structure; play-in just labels the first round differently when byes exist)
- “Seeding source”: Standings or Manual
- “Reseed each round”: toggle (default OFF). If OFF, bracket is fixed.
- “Lock bracket”: toggle (when ON, standings changes must not reshuffle seeds)
- Button: “Generate / Regenerate Bracket”
- Button: “Reset Playoffs” (danger action)

2) Bracket Rules (Single Elimination)
Given qualifyCount X:
- Compute bracketSize B = next power of 2 >= X (2,4,8,16,32,64,128).
- Byes = B - X.
- Seeds 1..X exist. Seeds > X are empty and represent BYEs.
- Seeding placement should follow standard bracket seeding.
Examples:
- X=4 => 1v4, 2v3
- X=8 => 1v8, 2v7, 3v6, 4v5
- X=12 => B=16 => seeds 1–4 get byes. Round 1 should be 5v12, 6v11, 7v10, 8v9. Then winners advance to face seeds 1–4 in round 2.
- X=48 => B=64 => 16 byes (seeds 1–16 get byes), Round 1: 17v48, 18v47, … etc with standard pairing.

3) Auto-Fill From Standings
If seeding source is “Standings”:
- Pull current standings for the division.
- Take top X teams (based on standings rank).
- Assign them to seeds 1..X.
- Store seeds in DB.
If locked = true, do not change seeds even if standings update.

If seeding source is “Manual”:
- Admin can select team for each seed slot via dropdown.

4) Overrides (Provision to change)
On the bracket admin page, allow:
- Editing any seed slot (seed -> team dropdown)
- Swapping seeds (drag and drop is nice; if not, provide a “Swap” UI)
- Editing game schedule fields: date/time, field/venue
- Lock bracket (prevents auto-refill)
