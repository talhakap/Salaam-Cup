CURRENT PROBLEM:
When admin approves a team, the system generates a username and password and displays it to the admin. The admin manually sends credentials to the captain.

This must be removed and replaced with a secure automated activation link flow.

GOAL:
When admin approves a team:
- Automatically generate a secure one-time activation token
- Email the captain a link to set their password
- Never generate or display passwords
- Never email passwords
- Token expires in 30 minutes
- Token is single-use

SECURITY REQUIREMENTS:
- Use crypto.randomBytes(32) for token generation
- Hash token using SHA-256 before storing
- Store only token_hash in database
- Use bcrypt (or existing hashing method in project) for password hashing
- Validate expiration and single-use logic

DATABASE:
If an activation_tokens table does not exist, create it:

activation_tokens:
- id (uuid)
- team_id (uuid)
- captain_email (text)
- token_hash (text)
- expires_at (timestamp)
- used_at (timestamp nullable)
- created_at (timestamp default now())

FLOW TO IMPLEMENT:

1) Modify existing team approval logic ONLY:
   - When approved = true
   - Generate rawToken
   - Hash token
   - Store token_hash + expires_at
   - Send email to captain_email with:
     ${APP_URL}/activate?token=RAW_TOKEN

2) Add backend route:
   POST /api/auth/activate
   Body: { token, password }

   Logic:
   - Hash incoming token
   - Find activation_tokens where:
       used_at IS NULL
       expires_at > now()
   - If valid:
       - Create or update captain user
       - Hash password
       - Link user to team
       - Mark used_at = now()
   - Return success or appropriate error

3) Add frontend page:
   /activate
   - Read token from URL
   - Show password + confirm password form
   - Submit to backend
   - Show success / expired / invalid messages
